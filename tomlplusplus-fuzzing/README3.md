# toml++ Fuzzing

This directory contains fuzzing experiments targeting the **toml++** C++ TOML parser.

The fuzzing setup was built using **libFuzzer**, **AddressSanitizer (ASan)**, and **UndefinedBehaviorSanitizer (UBSan)**.

The goal of this work was to explore the robustness of toml++ against malformed inputs and identify potential undefined behavior or crash conditions.

---

# Fuzzing Setup

All fuzzers were compiled with:

clang++ -O1 -g -DNDEBUG -std=c++17 \
-fsanitize=fuzzer,address,undefined \
-fno-omit-frame-pointer

Sanitizers used:

- AddressSanitizer (ASan)
- UndefinedBehaviorSanitizer (UBSan)

---

# Implemented Harnesses

The following fuzzing harnesses were implemented:

Harness | Purpose
------ | ------
harness_parse_toml.cpp | Fuzz the main TOML parser
harness_parse_toml_format.cpp | Parse then format back to TOML
harness_parse_json.cpp | Parse then convert to JSON using json_formatter
harness_parse_yaml.cpp | Parse then convert to YAML using yaml_formatter
harness_parse_at_path.cpp | Parse then exercise at_path() traversal logic

Each harness attempts to exercise different code paths inside the library.

---

# Corpus

Several seed corpora were created to guide the fuzzing process:

- corpus/ — basic TOML parser inputs  
- corpus_format/ — formatting related inputs  
- corpus_json/ — JSON conversion inputs  
- corpus_yaml/ — YAML conversion inputs  
- corpus_at_path/ — inputs targeting path traversal APIs  

These inputs contain small valid TOML examples to help libFuzzer explore deeper logic in the parser.

---

# Bugs Discovered

During fuzzing two independent bug families were discovered.

## Bug 1 — Parser Undefined Behavior

Malformed TOML inputs can trigger undefined behavior in the parser implementation.

Sanitizer output indicates an invalid virtual call involving utf8_reader_interface inside:

target/include/toml++/impl/parser.inl

Typical sanitizer report:

runtime error: member call on address which does not point to an object of type 'toml::impl::utf8_reader_interface'

This occurs instead of rejecting malformed TOML with a clean parse_error.

---

## Bug 2 — Unicode Classification Undefined Behavior

Malformed UTF-8 sequences can trigger execution of an unreachable code path inside:

target/include/toml++/impl/unicode_autogenerated.hpp

Example minimized input:

\xcc\xaa

UBSan report:

runtime error: execution reached an unreachable program point

Additional minimized inputs such as:

\xcf\x9b

also trigger the same issue.

---

# Status

Both bugs were discovered during fuzz testing and reported upstream.

Fuzzing was paused after discovering the issues to avoid repeatedly rediscovering the same bug families.

Further fuzzing may continue once upstream fixes are available.
